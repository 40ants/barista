# Lessons Learned

## Уроки за 2026-06-06

- **Контекст:** Порт Barista с LispWorks на SBCL с CFFI-мостом к ObjC.
- **Проблема/Ошибка:** `trivial-main-thread` перестаёт доставлять задачи после запуска `[NSApp run]` — главный поток занят AppKit и не дренирует очередь.
- **Решение:** Вместо `trivial-main-thread/call-in-main-thread` использовать GCD: `dispatch_async_f` с `_dispatch_main_q`.
- **Усвоенный урок:** После `[NSApp run]` единственный надёжный способ выполнить код на главном потоке — GCD main queue. `trivial-main-thread` подходит только для однократной передачи управления до старта AppKit.

- **Контекст:** Регистрация обработчика кликов на `NSStatusItem`.
- **Проблема/Ошибка:** `setAction:`/`setTarget:` установлены на `NSStatusItem`, но callback получает `NSStatusBarButton` в качестве `sender`, а не `NSStatusItem`. Попытка вызвать `(send sender "button" ...)` на кнопке падала с `unrecognized selector`.
- **Решение:** Устанавливать `setAction:`/`setTarget:` на `button` (через `[nsStatusItem button]`), ключевать `*click-table*` по адресу кнопки; в callback передавать `sender` напрямую как `inView:`.
- **Усвоенный урок:** AppKit передаёт в action `sender` тот объект, на котором установлен `setAction:`. Для `NSStatusItem` это всегда `NSStatusBarButton`, не сам `NSStatusItem`.

- **Контекст:** Передача `*plugin*` в меню и воркер-треды.
- **Проблема/Ошибка:** На главном потоке AppKit `*plugin*` равен NIL; меню-танки и колбеки не знали, к какому плагину они относятся.
- **Решение:** Захватывать экземпляр плагина в замыкание при создании танка и явно перебиндить `*plugin*` при каждом вызове: `(let ((*plugin* p)) ...)`. В воркер-тредах передавать через `:initial-bindings`.
- **Усвоенный урок:** Динамические переменные не наследуются автоматически ни в GCD-блоках, ни в `bordeaux-threads`. Всегда явно захватывать и перебиндить контекст.

- **Контекст:** `log:info` с тильдой в строке пути.
- **Проблема/Ошибка:** `(log:info "Loading plugins from ~/.config/...")` крашился — log4cl передаёт строку в `format`, который интерпретирует `~` как директиву.
- **Решение:** `(log:info "Loading plugins from ~A" dir)` — путь как аргумент `~A`.
- **Усвоенный урок:** В `log:info`/`log:error` и подобных макросах первый аргумент — format-строка. Любой литерал с `~` в нём должен быть вынесен в аргумент через `~A`.

- **Контекст:** `dispatch_get_main_queue()` через CFFI.
- **Проблема/Ошибка:** `dispatch_get_main_queue` — это C-макрос, не экспортируемая функция; `cffi:foreign-funcall` на ней не работает.
- **Решение:** Использовать `cffi:foreign-symbol-pointer "_dispatch_main_q"` — глобальная переменная, которую макрос адресует напрямую.
- **Усвоенный урок:** Перед биндингом C-API проверять, является ли символ настоящей функцией или макросом. macOS GCD-макросы (`dispatch_get_main_queue`, `dispatch_get_global_queue`) — адреса глобальных переменных, а не вызовы.

- **Контекст:** `stop` в pomodoro вызывался из воркер-треда и из меню.
- **Проблема/Ошибка:** Колбек меню вызывал `(funcall cb)` без аргументов, а `stop` был определён как `(defun stop (*plugin*) ...)` (или без `&optional`), что приводило к arity-ошибке.
- **Решение:** `(defun stop (&optional self) (declare (ignore self)) ...)` — функция принимает необязательный аргумент.
- **Усвоенный урок:** Функции, используемые одновременно как колбеки меню (0 аргументов) и как прямые вызовы (возможно с аргументом), должны принимать `&optional` параметр или быть обёрнуты лямбдой при регистрации.

- **Контекст:** `ros dump executable app.ros` в `build.sh`.
- **Проблема/Ошибка:** Команда всегда создаёт файл с именем `app` (имя скрипта без расширения), игнорируя ожидаемое имя `barista`.
- **Решение:** После дампа добавить `mv app barista`.
- **Усвоенный урок:** `ros dump executable <script.ros>` именует выходной файл по имени скрипта, не по желаемому имени бинаря. Явный `mv` обязателен.

- **Контекст:** Заголовок `.ros`-скрипта для Roswell.
- **Проблема/Ошибка:** Неправильное расположение `exec` в shebang-блоке приводило к тому, что скрипт не запускался как исполняемый.
- **Решение:** `exec` должен находиться внутри блока `#| ... |#`, причём закрывающий `|#` — на отдельной строке; использовать `$0`, а не `"$0"`.
- **Усвоенный урок:** Roswell `.ros`-скрипты имеют строгий формат shebang-блока. Отклонения от шаблона из документации ломают запуск без очевидных сообщений об ошибке.

- **Контекст:** `NSFontAttributeName` / `NSForegroundColorAttributeName` в CFFI.
- **Проблема/Ошибка:** Обращение к этим именам как к функциям или константам на этапе компиляции падало — в C они экспортируются как глобальные переменные типа `NSString *`, а не как `#define`.
- **Решение:** Получать значения через `cffi:foreign-symbol-pointer` после загрузки AppKit; разыменовывать как `:pointer`.
- **Усвоенный урок:** AppKit-константы вида `NSFooAttributeName` — глобальные переменные-указатели, инициализируемые только после загрузки фреймворка. Обращаться к ним только после `load-objc-frameworks`, через `cffi:foreign-symbol-pointer`.

## Уроки за 2026-06-11

- **Контекст:** `hide` удаляет `NSStatusItem` из статус-бара при остановке плагина.
- **Проблема/Ошибка:** `hide` вызывал `(send ns-item "button")` повторно — уже после `removeStatusItem:`/`release`. AppKit мог вернуть адрес, переиспользованный для кнопки другого живого плагина; `remhash` по этому адресу удалял чужую запись из `*click-table*`, и соседний плагин физически исчезал из menubar.
- **Решение:** Сохранять `(cffi:pointer-address button)` в слоте `button-address` объекта `status-item` в момент инициализации. В `hide` читать этот кешированный адрес — никакого повторного `send`.
- **Усвоенный урок:** Никогда не запрашивать дочерние ObjC-объекты (button, subview) у уже освобождённого или удалённого родителя. Адреса ObjC-объектов, полученные после `release`, ненадёжны и могут совпасть с живыми объектами. Кешировать нужные указатели/адреса до момента освобождения.

- **Контекст:** Редактирование `:import-from` в `defpackage` при добавлении нового символа.
- **Проблема/Ошибка:** При добавлении `#:get-button-address` в `:import-from #:barista/classes` был случайно удалён уже существующий `#:get-string-form-for-macro`. Сборка упала с `BARISTA/MENU::GET-STRING-FORM-FOR-MACRO is undefined` только на этапе компиляции плагинов.
- **Решение:** Восстановить пропущенный символ в `:import-from`.
- **Усвоенный урок:** При редактировании блока `:import-from` всегда сверяться с оригинальным содержимым — `oldString` в Edit должен захватывать весь блок целиком, чтобы исключить случайное удаление соседних строк.
